1. Introducción y Motivación 🚀

Bienvenida al curso y objetivos de aprendizaje.

Importancia del Desarrollo Frontend 🌐

El frontend es la cara visible de las aplicaciones web, brindando experiencias interactivas y atractivas a los usuarios.

Un buen desarrollo frontend mejora la usabilidad y satisfacción del usuario.

Estado del Arte en Desarrollo Frontend 🌐

El ecosistema del desarrollo frontend ha evolucionado significativamente en los últimos años, con varias librerías y frameworks que lideran la industria:

Angular

  • Desarrollado por Google
  • Framework completo basado en TypeScript
  • Ideal para aplicaciones de gran escala

React

  • Desarrollado por Facebook
  • Biblioteca para interfaces de usuario
  • Enfoque declarativo y virtual DOM

Vue.js

  • Creado por Evan You
  • Combina lo mejor de Angular y React
  • Fácil de aprender y adaptable

Estos frameworks y bibliotecas han establecido estándares y prácticas que han mejorado la eficiencia y la experiencia del desarrollador.

Popularidad de React 📈

Según la encuesta de State of JS:

  • React es la biblioteca de frontend más utilizada.

¿Por qué React? 🤔

React es una biblioteca JavaScript para construir interfaces de usuario.

  • Componentes reutilizables 🧩
  • Actualizaciones eficientes con Virtual DOM ⚡
  • Gran comunidad y ecosistema 🌐
  • Alta demanda en el mercado laboral 💼

Virtual DOM en React 🌳

¿Qué es?

  • Representación ligera del DOM real en memoria
  • Estructura de árbol de objetos JavaScript
  • Sincronizado con el DOM "real"

Ventajas

  • Mejora el rendimiento de la aplicación
  • Minimiza las manipulaciones del DOM
  • Actualiza eficientemente solo lo necesario

Instalación de Node.js y NPM 🛠️

Antes de empezar, necesitamos instalar Node.js y NPM:

  1. Visita nodejs.org y descarga la versión LTS (Long Term Support).
  2. Sigue las instrucciones de instalación para tu sistema operativo.
  3. Verifica la instalación abriendo una terminal y ejecutando: 
    
node -v
npm -v

Deberías ver las versiones instaladas de Node.js y NPM.

Creando una Aplicación React con Create React App 🚀

Create React App nos permite iniciar un proyecto de React rápidamente:

  1. Abre una terminal y navega hasta el directorio donde deseas crear tu proyecto.
  2. Ejecuta el siguiente comando: 
    
npx create-react-app my-app --template typescript

    Esto creará una nueva carpeta llamada mi-app con la estructura del proyecto.

  3. Ingresa al directorio del proyecto: 
    
cd my-app
  4. Inicia la aplicación: 
    
npm start

    La aplicación se abrirá en http://localhost:3000.

Declarativo vs Imperativo 🤔

Enfoques de programación en React.

Programación Imperativa 🛠️

Le decimos al programa cómo hacer las cosas paso a paso.


const numeros = [1, 2, 3];
const dobles = [];
for (let i = 0; i < numeros.length; i++) {
  dobles.push(numeros[i] * 2);
}
console.log(dobles); // [2, 4, 6]

Programación Declarativa 📝

Le decimos al programa qué queremos lograr.


const numeros = [1, 2, 3];
const dobles = numeros.map(n => n * 2);
console.log(dobles); // [2, 4, 6]

Componentes en React 🧩

Construyendo bloques reutilizables de UI.


function Saludo({ nombre }) {
  return <h1>Hola, {nombre}!</h1>;
}

export default function App() {
  return (
    <div>
      <Saludo nombre="Mundo" />
      <Saludo nombre="React" />
    </div>
  );
}

Componentes de Clase vs Funcionales 🧩

Comparación de sintaxis y uso.

Componente de Clase:


class Saludo extends React.Component {
  render() {
    return <h1>Hola, {this.props.nombre}!</h1>;
  }
}

Componente Funcional:


function Saludo({ nombre }) {
  return <h1>Hola, {nombre}!</h1>;
}

Estado en React 🎛️

El estado es información mutable que afecta cómo se renderiza el componente.

Permite crear interfaces interactivas y dinámicas.

Contraejemplo: Contador sin useState ❌

Un contador que no funciona correctamente:


function ContadorMalo() {
  let contador = 0;

  function incrementar() {
    contador += 1;
    console.log(contador);
  }

  return (
    <div>
      <p>Has hecho clic {contador} veces</p>
      <button onClick={incrementar}>
        Incrementar
      </button>
    </div>
  );
}

El valor de contador no cambia en la interfaz. ¿Por qué?

Uso Correcto del Estado ✅

Utilizando useState para manejar el estado:


function Contador() {
  const [contador, setContador] = useState(0);

  function incrementar() {
    setContador(contador + 1);
  }

  return (
    <div>
      <p>Has hecho clic {contador} veces</p>
      <button onClick={incrementar}>
        Incrementar
      </button>
    </div>
  );
}

JSX y TSX 🖋️

Sintaxis que facilita la creación de componentes de UI:

  • JSX: Sintaxis similar a HTML en JavaScript.
  • TSX: JSX con TypeScript para tipado estático.

Ejemplo de JSX 📝

Creando una lista de elementos:


function Lista({ items }) {
  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>{item.texto}</li>
      ))}
    </ul>
  );
}

Comparación entre JSX y TSX 📝

JSX y TSX son extensiones de sintaxis para JavaScript y TypeScript, respectivamente.

JSX

  • Extensión de JavaScript.
  • Permite escribir código similar a HTML dentro de JavaScript.
  • No incluye tipado estático.
  • Usa la extensión de archivo .jsx.

function Saludo(props) {
  return <h1>Hola, {props.nombre}!</h1>;
}

TSX

  • Extensión de TypeScript.
  • Combina JSX con tipado estático de TypeScript.
  • Ayuda a prevenir errores en tiempo de compilación.
  • Usa la extensión de archivo .tsx.

interface Props {
  nombre: string;
}

const Saludo: React.FC<Props> = ({ nombre }) => {
  return <h1>Hola, {nombre}!</h1>;
};

Ambos permiten crear componentes de React, pero TSX ofrece las ventajas del tipado estático de TypeScript.

Props y Estado 🎛️

Gestionando datos en los componentes:

  • Props: Parámetros de componentes (inmutables).
  • Estado: Datos internos del componente (mutables).

Hooks Básicos 🪝

Funciones que permiten usar estado y otras características en componentes funcionales:

  • useState
  • useEffect
  • useRef
  • useContext

Ejemplo de useEffect 🪝

Actualizando el título del documento:


function ContadorConEfecto() {
  const [contador, setContador] = useState(0);

  useEffect(() => {
    document.title = \`Has hecho clic \${contador} veces\`;
  }, [contador]);

  return (
    <div>
      <p>Has hecho clic {contador} veces</p>
      <button onClick={() => setContador(contador + 1)}>
        Incrementar
      </button>
    </div>
  );
}

Ejemplo de useRef 🪝

Accediendo a referencias del DOM:


function EnfoqueInput() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus();
  }, []);

  return <input ref={inputRef} />;
}

Introducción a useContext 🪝

Compartiendo datos globales sin pasar props manualmente.


// Creando un contexto
const ContextoTema = React.createContext('light');

// Componente que consume el contexto
function BotonTema() {
  const tema = useContext(ContextoTema);
  return <button className={tema}>Botón con tema {tema}</button>;
}

// Usando el contexto en App
function App() {
  return (
    <ContextoTema.Provider value="dark">
      <BotonTema />
    </ContextoTema.Provider>
  );
}
